## Hermes vs OpenClaw / Claude Code / Codex (official-source-only notes)

Goal: identify small, safe Hermes improvements inspired by mainstream agent workflows.

### Codex (OpenAI)

Notable public capabilities (official docs):

- Codex CLI is a terminal coding agent that can read/modify/run code; it supports multiple interaction modes (e.g. suggest/auto-edit/full-auto) and an in-session `/mode` toggle. (OpenAI Help Center)
- Codex is trained to provide verifiable evidence via citations of terminal logs and test outputs. (OpenAI system card addendum / product post)

References (official):

- https://help.openai.com/en/articles/11096431-openai-codex-ligetting-started
- https://openai.com/index/introducing-codex/
- https://cdn.openai.com/pdf/8df7697b-c1b2-4222-be00-1fd3298f351d/codex_system_card.pdf

Small Hermes opportunities:

- Make “evidence” a first-class UX concept in the CLI: a consistent, compact “evidence panel” (tests run, key tool outputs, files changed) at the end of a run, even when streaming.

### Claude Code (Anthropic)

Notable public capabilities (official docs):

- Centralized permissions model with `allow`/`deny` rules in settings, plus workflows to view/manage rules (e.g. `/permissions`). (Anthropic docs)
- Hook system to intercept/extend agent behavior at defined lifecycle events. (Anthropic docs)

References (official):

- https://docs.anthropic.com/en/docs/claude-code/overview
- https://docs.anthropic.com/en/docs/claude-code/settings
- https://docs.anthropic.com/en/docs/claude-code/hooks
- https://docs.anthropic.com/en/docs/claude-code/iam

Small Hermes opportunities:

- Improve discoverability for hooks: a `hermes hooks` helper that prints current hook config + last hook run status (CLI-only, no behavior change).
- Add a documentation snippet for “deny always beats allow” + common secure defaults (`.env`, SSH keys, cloud creds) in Hermes’ config/hardening docs.

### OpenClaw

Notable public capabilities (official docs):

- Tools are treated as typed function definitions; plugins can register additional tools. (OpenClaw docs)
- Tool profile concept where “coding” includes lightweight web tools but not full browser automation by default. (OpenClaw docs)

References (official/public docs):

- https://docs.openclaw.ai/tools
- https://docs.openclaw.ai/browser

Small Hermes opportunities:

- Offer a “web-light” toolset preset for safer default deployments (web_search/web_fetch equivalents but no browser automation), mirroring the profile idea for quick hardening.
